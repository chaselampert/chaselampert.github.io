Predicting the number of wins and playoff contention in the National Football league¶

Chase Lampert¶

Intro: Team data from the National Football League from the years 2010 through 2016 will be used alongside some data science principles to try to predict the number of wins in a season for an NFL team. We will also try and predict if a team will be in playoff contention. The NFL is an economic giant, producing over 10 billion dollars in 2016 revenue. Data science can be a key an integral part of predicting which teams will perform well as well as individual games. If a person were to make an accurate enough model they would be in a very advantageous position. This is one reason to learn more about the potential of data science and the NFL in this tutorial.

Concentrating the data into one table:¶

This website provides the nessesary data, but it is all in seperate tables. The link above brings you to a web page where it lists the years on the left hand side. Start by clicking on the year 2016 (the 2017 season is not complete). Under the header "AFC standings" click on the "share and more tab" this opens a drop down menu, click on "get table as csv". The format of the table changes from a table to a csv, highlight the table and copy it. Paste the table into a text editor. Repeat this proccess for "NFC standings". After you have gotten both the AFC and NFC table, repeat this proccess for the years 2015 though 2010, pasting all of the CSV tables into the same text editor page. Once you have years 2010 through 2016 in a text editor, go into the text editor, use the find and replace tool to replace all commas with tab's (\t). Then copy everything in the text editor and paste it into either a numbers table if you're on a mac or an excel table if you are on another machine. Then download the table as a CSV and upload it into the same directory as your python project.

Explaining the data:¶

Tm: The NFL team that this row's data belongs to W: The number of wins out of 16 for that team during that season. L: The number of losses out of 16 for that team during that season. T: The number of ties out of 16 for that team during that season. W-L%: Win loss percentage for that team during that season. PF: Point for, the total number of points scored. PA: Points allowed, the total number of points allowed. PD: Point differential, the difference between points for and points allowed. MoV: Margin of Victory, the average amount of points a team won or lost by. SoS: Strength of Schedule, the average quality of opponent based on SRS SRS: Simple Rating System, the teams quality relative to the average (0.0) OSRS: Offensive Simple Rating System, the team's offensive quality relative to the average (0.0) DSDS: Defensive Simple Rating System, the teams defensive quality relative to the average (0.0)

There are still issues with the data. First there is an extra row where all of the values are NaN we can outright drop this row. Next we see that the DSRS value is NaN for the tail of our dataframe. This comes from when we curated our data. In years such as 2010, which is represented in the tail, there were no ties - thus the T column was not put into those tables. When we copy and pasted our data we did not take this into account. This means that for season in which there were on ties every column from W-L% and right is shifted to the left one column. To clean this data we will traverse through the entire dataframe and if DSRS is NaN then shift all values from column T on to the right one and set T equal to 0.

        df.iloc[idx-1] = row

The above two graphs both display that time, in general, does not affect statistics in the NFL. The first graph shows that the total points scored across the league has been consistent for the last 7 years. The first bar graph represents points allowed where the biggest differential between two years was around 700 points which is NOT significant. There are 32 teams in the league and all 32 teams play 16 games, thus for each year there are 512 inputs into points allowed. A difference of 700 points in a season just means that teams tended to score around one more point on average for games in that season. One point in football is not significant. The second bar graph is based on the total points for per season, again we do not see a major difference over time.

The scatter plot shows the number of wins by each team for each year. The reason that each year does not have 32 dots is that teams overlap with the number of wins. This graph shows us that time does not have an effect on the range or number of wins in an NFL season.

Machine Learning¶

Choosing our Hypothesis: In this section we are going to use machine learning to try and predict the number of wins by a team in an season. These predictions will be based off points for, points allowed, margin of victory, simple rating system, offensive SRS, and defensive SRS. The first machine learning algorithim will take these six statistics and output a predicted number of wins. This will be done by supervised learning which is the principle of splitting our data into a training set and a test set. We train the model with the test set by giving the model our six stats along with the win totals so it can determine how the stats correlate with wins. Then for the test set the model will only see the six stats and give a prediction for the number of wins. Then we will access the accuracy of our prediction against the actual values. The hypothesis is that the six values can accurately predict the number of wins by a team in a season.

Next we will do ten fold cross validation to ensure that we did not overfit the training data. In cross-validation we partition the original data into ten equal size subsamples. Of the ten samples one is used as testing or validation of the model. The remaining 9 subsamples are used as training data. This process is then repeated ten times with each subsample used once as the testing data. We will then average the ten results to produce a single number to show the accuracy of our model.

To learn more about overfitting: https://elitedatascience.com/overfitting-in-machine-learning

To learn more about cross validation: http://scikitlearn.org/stable/modules/generated/sklearn.model_selection.KFold.html.

Hypothesis testing¶

From our cross validation scores we can see that our model is not perfect at predicting the number of wins, but still does a decent job. The question we are seeking to answer is if there is a relationship between our six stats and the number of wins per team. To answer this we will use our R2 value. The R2 value compares the variation for all win values sharing common X values with variation from the prediction. The best possible R2 value is 1, while the worst would be 0. Our R2 value of .83 is very good and it means that our data explains most of the error found in our model.

For more on R2 values: http://scikit-learn.org/stable/modules/generated/sklearn.metrics.r2_score.html
